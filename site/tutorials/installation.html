Installation

CLAVIN (the core) is not designed to be a standalone component, rather bundled in another application of processing pipeline. A common use case is to integrate CLAVIN with Apache Hadoop or Twitter Storm.

This section will discuss mechanisms for quickly integrating CLAVIN with a JVM-based project.

Using CLAVIN via Maven

You will need to build an index before using CLAVIN. Please follow the instructions on Building a Gazetteer Index from Geonames to accomplish this task.

Reference the Berico Technologies Nexus Repository in your Maven pom.xml:

<repositories>
  <repository>
     <id>nexus.bericotechnologies.com</id>
     <name>Berico Technologies Nexus</name>
     <url>http://nexus.bericotechnologies.com/content/groups/public</url>
     <releases><enabled>true</enabled></releases>
     <snapshots><enabled>true</enabled></snapshots>
  </repository>
</repositories>

Add a dependency within your pom.xml to the CLAVIN project:

<dependency>
   <groupId>com.berico</groupId>
   <artifactId>clavin</artifactId>
   <version>0.4.0</version>
</dependency>

Building CLAVIN from source and referencing the JAR in Eclipse

In this example, we will checkout (or download) the code from GitHub, compile the project via Maven, and add it as a build dependency in Eclipse.

1a. Checkout the CLAVIN project from GitHub.

git clone https://github.com/Berico-Technologies/CLAVIN.git
cd CLAVIN

1b. Download the CLAVIN project as a ZIP file from GitHub.

The url location for the latest tagged branch should be: https://github.com/Berico-Technologies/CLAVIN/archive/release/0.4.0.zip.

wget https://github.com/Berico-Technologies/CLAVIN/archive/release/CLAVIN-release-0.4.0.zip
unzip CLAVIN-release-0.4.0.zip
cd CLAVIN-release-0.4.0

2. Compile and package CLAVIN.

mvn package

Hopefully, all the test will pass (like they should) and you will have a pristine JAR waiting for you in the target directory. If we screwed something up, and you want to temp fate, you are welcome to skip the tests:

mvn package -DskipTests

3. Acquire the compiled JAR.

In the target directory, you should have the following JARs:

  • clavin-0.4.0.jar - JAR with only the compiled classes and embedded resources.
  • clavin-0.4.0-jar-with-dependencies.jar - JAR with CLAVIN's dependencies bundled (a.k.a. Fat JAR).
  • clavin-0.4.0-sources.jar - JAR with the source files.
  • clavin-0.4.0-javadoc.jar - JAR with JavaDocs.

If you so desire, you can move these libraries to the location of your choice:

cp target/clavin-0.4.0-jar-with-dependencies.jar /my/library/location/

Where ever you keep these files, you will need to know the location to import the libraries into Eclipse.

4. Import CLAVIN into Eclipse as a User Library.

Assuming you have Eclipse started and you are importing CLAVIN into an existing project:

  1. Right click on the project, select Build Path > Add Libraries.
  2. In the Add Library window, select User Library and click Next.
  3. Click New in the User Libraries window.
  4. Name the library (like "CLAVIN"), and press OK.
  5. Select the new library, and press Add External JARs.
  6. Browse for the clavin-0.4.0-jar-with-dependencies.jar file.
  7. Select the the clavin-0.4.0-jar-with-dependencies.jar file and click Open.
  8. (Optional) Import the Source JAR - expand the CLAVIN JAR, select Source attachment and press the Edit button on the right. Select External location and press External File. Navigate to the clavin-0.4.0-sources.jar file and click Open and then press OK.
  9. (Optional) Import the JavaDoc JAR - select Javadoc location and press the Edit button on the right. Select Javadoc in archive, select External file and press Browse to the right of the Archive path textbox. Navigate to the clavin-0.4.0-javadoc.jar file and click Open and then press OK.
  10. Verify the results, and press OK. Check the box next to the library, and then press Finish.

If you are having trouble following along, the process should look something like this (though these screenshots were taken on a Mac with SpringSource Tool Suite):

1. Add Libraries 2. Add User Library 3. Add New User Library 4. Name the Library
5. Add External JAR 6. Browse for the JAR. 7. Import the compiled JAR. 8. Import Source JAR.
9. Import JavaDoc JAR. 10. Verify and Finish.    
   

5. Youre finished!

You should now be able to use CLAVIN with Eclipse.
